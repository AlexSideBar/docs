## Getting the Best Support

<Steps>
  <Step title="Check Documentation">
    Search our documentation first - your question might already be answered.
    
    Use **Command + K** to open the AI search assistant for instant access to documentation answers.
  </Step>

  <Step title="Search Existing Issues">
    Check GitHub Issues and Discord channels for similar problems and solutions.
  </Step>

  <Step title="Gather Information">
    Please provide the following details for efficient support:
    
    * Alex Sidebar version (e.g., 1.3.7)
    * Xcode version (e.g., 16.2)
    * Steps to reproduce (if reporting an issue)
  </Step>
</Steps>

## Reporting Issues

<Steps>
  <Step title="Create a GitHub Issue">
    Visit our [bug report page](https://github.com/DanielEdrisian/AlexSideBar-Public/issues/new?assignees=DanielEdrisian&labels=bug&projects=&template=bug-report.md&title=), 
    which will automatically use our bug report template.
  </Step>

  <Step title="Include Essential Information">
    ```markdown
    **System Information:**
    - Alex Sidebar Version: x.x.x
    - macOS Version: xx.xx
    - Xcode Version: xx.x

    **Description:**
    Clear description of the issue

    **Steps to Reproduce:**
    1. Step one
    2. Step two
    3. Step three

    **Expected Behavior:**
    What should happen

    **Actual Behavior:**
    What actually happens

    **Screenshots/Recordings:**
    If applicable
    ```
  </Step>

  <Step title="Recording Issues">
    For visual issues or complex workflows:
    1. Use macOS Screen Recording (Shift + Command + 5)
    2. Keep recordings focused and under 2 minutes
    3. Upload to GitHub Issues or share via Discord
  </Step>
</Steps>

## Feature Requests

<Steps>
  <Step title="Create a Feature Request">
    Visit our [feature request page](https://github.com/DanielEdrisian/AlexSideBar-Public/issues/new?assignees=DanielEdrisian&labels=enhancement&projects=&template=feature-request.md&title=), 
    which will automatically use our feature request template.
  </Step>

  <Step title="Include Essential Information">
    ```markdown
    **Feature Description**
    A detailed description of the requested feature.

    **Screenshots/Screen Recordings**
    Please attach any relevant visual materials.

    **Current Version**
    - Version [e.g., 1.2.0]
    ```
  </Step>

  <Step title="Gather Community Support">
    Share your idea on Discord to gather feedback and discuss implementation possibilities.
  </Step>
</Steps>
