## Getting the Best Support

### Before Asking for Help

<Steps>
  <Step title="Check Documentation">
    Search this documentation first - your question might already be answered!
    You can use Command + K to open the AI search assistant which will help
    answer your questions and queries directly from the documentation.
  </Step>

<Step title="Search Existing Issues">
  Check GitHub issues and Discord channels for similar problems and solutions.
</Step>

  <Step title="Gather Information">
    Be ready to provide: - Alex Sidebar version - macOS version - Xcode version
    - Steps to reproduce (if reporting an issue)
  </Step>
</Steps>

## Reporting Issues

### How to Submit a Bug Report

<AccordionGroup>
  <Accordion title="1. Create a GitHub Issue">
    Visit our [GitHub Issues](https://github.com/DanielEdrisian/AlexSideBar-Public/issues) page
    and click "New Issue"
  </Accordion>

  <Accordion title="2. Include Essential Information">
    ```markdown
    **System Information:**
    - Alex Sidebar Version: x.x.x
    - macOS Version: xx.xx
    - Xcode Version: xx.x

    **Description:**
    Clear description of the issue

    **Steps to Reproduce:**
    1. Step one
    2. Step two
    3. Step three

    **Expected Behavior:**
    What should happen

    **Actual Behavior:**
    What actually happens

    **Screenshots/Recordings:**
    If applicable
    ```

  </Accordion>

  <Accordion title="3. Recording Issues">
    For visual issues or complex workflows:
    1. Use macOS Screen Recording (Shift + Command + 5)
    2. Keep recordings focused and under 2 minutes
    3. Upload to GitHub issue or share via Discord
  </Accordion>
</AccordionGroup>
